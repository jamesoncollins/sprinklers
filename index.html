How to Use / Pressure Guide

Plan a sprinkler layout

  1. Set the site. Use the sketch grid, upload an image and calibrate a known distance, or look up an address for satellite imagery.
  2. Load catalog data. Use the built-in catalog or import CSV manufacturer rows, then choose the head, nozzle, and rated catalog pressure.
  3. Create zones. Add one zone per valve/runtime group and enter static pressure, optional dynamic pressure, open-flow supply, and water share.
  4. Place sprinklers. Click the canvas to add heads, assign each to a zone, then set arc, rectangular throw, and orientation.
  5. Review analysis. Use the cards and overlays to compare operating pressure, actual flow, throw, and precipitation before saving the project.

Water pressure model

Static pressure
No-flow gauge pressure available to the zone.
Dynamic pressure
Optional measured pressure while the zone is running. When entered, it overrides the calculated pressure and is capped at static pressure.
Open-flow supply
Estimated source flow at 0 PSI. The app uses it with static pressure to solve the supply/demand balance.
Water share
Runtime multiplier for zones that should intentionally water longer or shorter than the base schedule.

When no dynamic pressure is entered, the app calculates pressure drop by comparing static pressure to the operating pressure found at the source/head balance. The source curve starts at open-flow GPM at 0 PSI and falls to 0 GPM at static pressure:

supply = open-flow × sqrt(1 - operating ÷ static)

For each trial pressure, sprinkler demand is the sum of every active head's pressure-adjusted flow. The app uses bisection between 0 PSI and static pressure until supply and demand intersect; the pressure drop shown in analysis is static - operating.

Unregulated heads scale flow and throw by sqrt(operating ÷ rated). Pressure-regulated heads use that same curve below their regulator pressure and stop gaining flow/throw above the regulator setting.

Precipitation results

Precipitation uses solved pressure, actual flow, and actual coverage geometry instead of treating manufacturer PR columns as authoritative:

PR = 96.3 × flow GPM ÷ irrigated sq ft

Catalog precipitation values are shown as nominal references because they may assume square or triangular spacing layouts.

Point-map precipitation is not uniform inside the throw. Arc and rotor heads use the default radial spreading profile S(ρ) = 1 / (1 + (ρ / 0.287)^2.48), where ρ is distance divided by throw radius. The 0.287 spreading factor controls how quickly water tapers with distance from the head; the profile is normalized so the map still integrates back to the solved head flow. Rectangular throws use a normalized 1 / distance spreading model with a minimum distance of 8% of the larger rectangle dimension to avoid an infinite spike at the head.
